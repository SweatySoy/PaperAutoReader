## Session Summary

### Task
作为"具备极高学术品味的资深 AI 软件架构师"，搭建 `src/models.py` 和 `src/config_loader.py`，
并实现 Filter Agent 的骨架代码。

### Completed Deliverables

#### 1. `src/models.py` - Pydantic 数据模型
实现了论文生命周期状态模型:
- `CandidatePaper`: Search Agent 输出 / Filter Agent 输入
- `ScoredPaper`: Filter Agent 输出 / Analysis Agent 输入 (含双轴评分)
- `AnalyzedPaper`: Analysis Agent 输出 / Report Agent 输入 (含深度分析)
- `FinalReport`: Report Agent 输出 (最终报告)
- `QuadrantCategory`: 四象限枚举 (CROWN_JEWEL, CORE_TRACK, IMPACT_TRACK, REJECTED)

#### 2. `src/config_loader.py` - YAML 配置加载器
- 单例模式实现
- 自动发现 `fields/Domain_Profile_*.yaml` 配置文件
- 提供类型安全的配置访问接口
- 支持时间衰减参数、评分阈值、关键词列表、实体列表等

#### 3. `src/filter_agent.py` - Filter Agent 骨架
- `TimeDecayCalculator`: 时间衰减权重计算
- `CoreScoreCalculator`: Core Score 计算 (语义匹配 + 关键词 + 任务相关性)
- `ImpactScoreCalculator`: Impact Score 计算 (时间衰减权重融合)
- `QuadrantRouter`: 四象限路由
- `FilterAgent`: 主编排器
- 抽象接口协议 (`EmbeddingService`, `LLMScoringService`)

#### 4. `src/test_filter.py` - 测试脚本
- 5篇 Mock 论文数据
- 覆盖四象限场景
- 验证时间衰减逻辑

### Strict Constraints Compliance
1. **绝对解耦**: 代码中无任何硬编码的学科/期刊/机构信息
2. **数据验证**: 全部使用 Pydantic 模型
3. **接口抽象化**: LLM/Embedding 调用使用 Protocol 接口预留
4. **核心逻辑**:
   - 时间衰减权重: 新论文(50% Venue + 30% Author + 20% Github), 老论文(20% Venue + 80% Citation)
   - 二维象限映射: 依据阈值判定四象限
